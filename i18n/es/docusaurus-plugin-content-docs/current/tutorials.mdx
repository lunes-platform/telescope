# 📝 Tutoriales

## Cómo funciona Lunes Node

Lunes Blockchain es una red descentralizada y distribuida basada en el algoritmo de consenso LPoS. Esta red puede ser mainnet o testnet, en mainnet los nodos funcionan con valores reales de tokens, monedas, NFT y activos. Por otro lado, testnet tiene tokens "falsos" para proporcionar un entorno de prueba.
En el lenguaje de desarrollo, mainnet es prod y testnet es dev.

Lunes Node es un node o servidor que tiene la responsabilidad de mantener y validar la blockchain, transmitir nuevas transacciones y proporcionar una API para la interacción e integración con la red.

Con un Lunes Node, se convierte en un participante de Lunes Blockchain y puede proporcionar resistencia a la red, validar datos e integrar servicios del mundo real en la blockchain.

## Cómo Instalar

<Tabs>
  <TabItem value="🐧 Linux" label="🐧 Linux" default>

  Necesitas instalar **[java8](http://www.google.com/search?q=how+to+install+java+8+linux)**

  Consigue tu versión de **[Lunes Node](./releases)**

  <details>
    <summary>Archivo de configuración Lunes Node</summary>

```yml title="lunesnode.conf"
# Mínimo necesario para funcionar
lunes {
    # ¿Dónde persistirá la blockchain?
    directory = "./"

    # Elija su chain MAINNET o TESTNET
    blockchain.type = ""

    # Ponga ip:port de cualquier node en chain que elija
    # MAINNET 5.196.155.34:7770
    # TESTNET 5.196.155.46:7770
    network.known-peers = [""]

    # Aquí se esconderá (encriptar) tu seed
    wallet.password = ""

    # Tu seed encriptado en BASED58
    wallet.seed = ""
  }
  ```

- Archivo de configuración completo **[here](https://github.com/lunes-platform/lunesnode/blob/master/src/main/resources/application.conf)**

</details>


  **Árbol de Archivos**

  ```js title="/path/mylunesblockchain/"
    ├── lunesnode-latest.jar
    └── lunesnode.conf
  ```

  **Para ejecutar**

  ```bash
  java -jar lunesnode-latest.jar lunesnode.conf
  ```

  **Si desea permitir que Lunes Node se ejecute para siempre, cree un servicio**

  <details>
    <summary> Archivo de configuración Systemd</summary>

  ```systemd title="/etc/systemd/system/lunesnode.service"
  [Unit]
  Description = Lunes Node Blockchain
  After = network.target

  [Service]
  WorkingDirectory = /path/mylunesblockchain/
  ExecStart = /usr/bin/java -jar lunesnode-latest.jar lunesnode.conf
  Restart = always
  RestartSec = 30s
  StandardOutput = journal
  StandardError = journal
  SyslogIdentifier = lunesnode

  [Install]
  WantedBy = multi-user.target
  ```

  </details>

<TabItem value="🪟 Windows" label="🪟 Windows">

  Tienes que instalar **[java8](https://www.java.com/pt-BR/download/)**

- Después de la instalación, verifique su versión de Java en la línea de comando **CMD**
  
  ```bash
  java -version
  ```

  Obtener una versión de **[Lunes Node](./releases)**

  Dentro de tu carpeta haz un archivo con un nombre **lunesnode.conf** con este contenido: 

  <details>
    <summary>Archivo de configuración Lunes Node</summary>

  ```yml title="lunesnode.conf"
  # Mínimo necesario para funcionar
  lunes {
    # ¿Dónde persistirá la blockchain?
    directory = "./"

    #  Elija su chain MAINNET o TESTNET
    blockchain.type = ""

    # Ponga ip:port de cualquier node en chain que elija
    # MAINNET 5.196.155.34:7770
    # TESTNET 5.196.155.46:7770
    network.known-peers = [""]

    # Aquí se esconderá (encriptar) tu seed
    wallet.password = ""

    # Tu seed encriptado en BASED58
    wallet.seed = ""
  }
  ```

  - Archivo de configuración completo **[here](https://github.com/lunes-platform/lunesnode/blob/master/src/main/resources/application.conf)**

  </details>

  **Árbol de Archivos**

  PATH = la ruta donde guardaste el archivo 

  ```js title="/PATH/mylunesblockchain/"
    ├── lunesnode-latest.jar
    └── lunesnode.conf
  ```

  - Después de obtener una versión de Lunes Node, en la línea de comand **CMD**, acceda a su carpeta cuando guarde estos archivos: 
 
  ```bash
  cd /PATH/mylunesblockchain
  ```
  
  **Para ejecutar**

  ```bash
  java -jar lunesnode-latest.jar lunesnode.conf
  ```

  **verificación de LOG **

  - In cmd, inside your folder, type it:

  ```bash
  cd PATH\mylunesblockchain\log
  ```

  **Si desea permitir que Lunes Node se ejecute para siempre, cree un servicio:**

  - En la carpeta de su node, cree un archivo con el nombre **node.bat** con este contenido:
  
  ```bash
  java -jar PATH\mylunesblockchain\lunesnode-latest.jar PATH\mylunesblockchain\lunesnode.conf
  ```

  - Descargar una versión 2.24 de NSSM (2014-08-31) **[here](http://nssm.cc/release/nssm-2.24.zip)**

  - Descomprimir el archivo

  - Abra el ***CMD*** como administrador

  - Verifique su versión de Windows 64 o 32 bits

    Para 64 bits escribe:
    
    ```bash
      cd PATH\nssm-2.24\win64
    ```

    Para 32 bits escribe:
    
    ```bash
      cd PATH\nssm-2.24\win32
    ```

  - Escriba los comandos a continuación

  ```bash
  nssm.exe install "LunesNode" "PATH\mylunesblockchain\node.bat"
  ```
  
  ```bash
  nssm set LunesNode DisplayName "LunesNode"
  ```

  ```bash
  nssm set LunesNode AppDirectory "PATH\mylunesblockchain"
  ```

  ```bash
  nssm set LunesNode Description "start lunes node with node.bat : java -jar PATH/lunesnode-latest.jar PATH/lunesnode.conf"
  ```

  ```bash
  nssm set LunesNode Start SERVICE_AUTO_START
  ```
